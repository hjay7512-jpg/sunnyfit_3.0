# 🏠 首页 (Home)

> **页面定位**：App 主 Tab「Home」下的首屏。顶栏固定不随滚动；主体为 Banner（可选）、今日状态/无计划引导、AI 建议卡片、My Progress、My Courses；底部为 Tab 栏与 AI 对话入口。各入口按业务跳转至个人中心、积分、消息、日历、器械、计划创建、锻炼、课程等模块。

---

### 🔝 顶栏 `[home_top_bar]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 页面常驻 | L1 核心 | 个人中心 / 积分中心 / 消息 / 日历 / 器械 |

**📝 业务逻辑**：
* **布局**：顶栏固定在视口顶部，不随内容滚动；与下方内容区保留合理间距。
* **左侧**：用户头像，点击进入个人中心（Profile）。
* **中部**：连续打卡天数展示（如「🔥 N 天」），点击进入积分/打卡中心。
* **右侧**：消息入口、日历入口、器械入口，点击分别进入消息列表、日历视图、设备连接/器械管理。
* **数据**：打卡天数来自用户连续打卡业务数据；头像来自当前用户资料。

---

### 📢 Banner `[home_banner]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 有配置时展示 | L2 运营 | 关闭 / 点击跳转（若配置链接） |

**📝 业务逻辑**：
* **展示条件**：仅当服务端或本地配置了首页 Banner 时展示；无配置时不占位。
* **内容**：图片与跳转链接由运营配置；无图时可使用占位样式。
* **关闭**：用户点击关闭按钮后，当次会话内不再展示（或按业务约定持久化关闭状态）。
* **布局**：与主体内容区同宽、对齐，不产生横向偏移。

---

### 📋 无计划状态 · 今日概览与行动引导 `[home_suggestion_no_plan]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 用户当前无进行中计划时展示 | L1 核心 | 创建计划（plan_create） |

**📝 业务逻辑**：
* **展示条件**：仅当用户「无进行中的计划」时展示本卡片。
* **今日概览**：展示当日汇总数据（如运动时长、消耗卡路里、跑步距离等），数据来自当日运动/健康统计；无数据时展示 0 与单位。
* **行动引导**：文案引导用户创建计划或尝试 Quick Start / Active Recovery；主 CTA「Try Now」进入创建计划流程（plan_create）。

---

### 📅 今日状态行（Day + Recovery）`[home_day_recovery]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 有进行中计划时展示 | L1 核心 | 仅展示，点击计划区域可进计划详情 |

**📝 业务逻辑**：
* **展示条件**：用户存在进行中计划时展示。
* **左侧**：当前计划所处「第 N 天」，N 来自计划进度业务数据。
* **右侧**：当日类型标签与卡路里目标（如「Recovery • 已消耗/目标 kcal」），数据来自当日计划与完成情况。
* **交互**：本行以信息展示为主；进入计划详情由 My Progress 中 Plan 卡片承担。

---

### ✨ AI 建议卡片 `[home_ai_suggestion]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 有 AI 推荐结果时展示 | L1 核心 | 开始锻炼 / 换一荐 / 快捷能力 |

**📝 业务逻辑**：
* **展示条件**：当存在 AI 生成的今日建议（推荐课程/训练等）时展示；无推荐可不展示或展示通用引导。
* **提示语**：由 AI 或运营配置的短文案，说明推荐理由或行动建议。
* **推荐内容**：一张主推荐（封面、标题、时长/难度等元信息）；来源可为课程、实景、动作训练、自由训练等。
* **Start Now**：主按钮，点击进入该推荐对应的锻炼/课程播放或详情页。
* **快捷 chips**：如「换一荐」「XX 分钟跑步」等，由 AI 或规则生成；点击可刷新推荐、或直接进入对应训练/能力。
* **数据**：推荐内容与 chips 来自推荐服务或 AI 接口，非前端写死逻辑。

### 📊 我的进度（My Progress）`[home_my_progress]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 横向滑动查看 | L1 核心 | 计划详情或创建 / 锻炼数据 |

**📝 业务逻辑**：
* **区域标题**：「My Progress」；内容区横向滚动，支持多块卡片。
* **Plan 卡片**：
  * 有进行中计划时：展示计划名称、当前阶段（如 Stage I～IV）及阶段进度条；当前阶段高亮；点击进入计划详情或计划 Dashboard。
  * 无计划时：展示「创建计划」引导，点击进入 plan_create。
* **Workout 卡片**：展示近 7 天锻炼汇总（如本周总时长、按日分布）；点击进入锻炼数据/历史页。
* **扩展**：可按业务增加 Fitness（如体重趋势）等卡片，数据来自健康/体重业务。

---

### 📚 我的课程（My Courses）`[home_my_course]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| Tab 切换 / 横向滑动 / View All | L1 核心 | 课程列表或详情 |

**📝 业务逻辑**：
* **区域标题**：「My Courses」。
* **子 Tab**：History（历史）、Favorites（收藏）、Redeemed（已兑换），切换后下方列表展示对应数据源。
* **课程列表**：当前 Tab 下的课程卡片，横向滚动；每卡展示封面、标题、时长与难度等；点击进入课程详情或播放。
* **View All**：进入该 Tab 对应的完整课程列表页。
* **数据**：列表来自用户历史/收藏/兑换记录接口，条数按业务约定（如最多首屏 N 条）。

---

### 🔘 底部 Tab 栏 `[home_bottom_tab]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 点击 Tab / 点击 AI 入口 | L1 核心 | 切换主模块 / 打开 AI 对话 |

**📝 业务逻辑**：
* **主 Tab**：Home、Workout、Challenge、Social 共用一个玻璃风格容器；当前所在 Tab 高亮（如 Home 为红色圆底）。
* **切换**：点击对应 Tab 切换至该主模块（本页即 Home 模块）。
* **AI 入口**：右侧独立入口（如 ✨ 图标），点击打开 AI 对话浮层或跳转至 AI 对话页；与主 Tab 视觉区分。
* **样式**：底部栏采用毛玻璃风格，不随内容滚动，始终固定在底部。

---

### 📌 路由与数据约定（汇总）

| 入口 | 目标页面/能力 |
| :--- | :--- |
| 顶栏头像 | 个人中心（profile） |
| 顶栏打卡 | 积分/打卡中心（points） |
| 顶栏消息 / 日历 / 器械 | 消息（message）、日历（calendar）、设备连接（device_connect） |
| 无计划 Try Now | 创建计划（plan_create） |
| AI 建议 Start Now / chips | 锻炼或课程播放/详情（workout 等） |
| My Progress · Plan | 计划详情或 plan_create |
| My Progress · Workout | 锻炼数据/历史 |
| My Courses · View All | 课程列表 |
| 底部 Tab | 切换 Home / Workout / Challenge / Social |
| 底部 AI 入口 | 打开 AI 对话 |

**📝 说明**：页面内不依赖前端写死的「模拟切换」「固定文案」作为业务规则；天数、推荐内容、列表数据等均应由业务接口或配置提供。

# 运动结果页 (Workout Result )

## 一、 核心架构：三态解耦状态机 (View Mode Machine)

为满足“即时结算爽感”、“历史数据沉淀”与“站外拉新分享”的差异化诉求，运动结果页作为统一的载体，通过传入 `view_mode` 参数，实现**一套代码、三种形态**的动态渲染：

### 1. 练后即时态 (`view_mode = immediate` / `record`)

* **场景**：用户刚刚完成运动，从锻炼引擎退出载入该页面。
* **主视觉**：触发“一镜到底”动效。页面初始由“火焰/奖杯徽章+一句话成就描述”居中展示，2.5 秒后图标和成就描述上移缩小，下方数据面板平滑滑出。若打破记录，数据项（如配速）展示红色 `BEST` 高光标签，且主视觉替换为奖杯。
* **转化交互 (强阻断)**：动效结束后，延迟弹出“AI 教练留存建议”底部抽屉（Bottom Sheet），覆盖原有的底部按钮。根据用户状态（破纪录/有无计划）动态展示对应AI教练建议卡片。
* **底部出口**：用户关闭 AI 建议抽屉后，露出常驻的 `[ Post My Result ]` 按钮，点击进入带有图片/文案校验的社区发帖流。

### 2. 历史查档态 (`view_mode = history`)

* **场景**：用户从“历史记录”或日历中点击进入，作为“数字日记”回顾。
* **主视觉**：**禁用**所有庆祝动效和一镜到底转场。页面无延迟直接展示最终的静态数据面板，保留成就徽章（火焰/奖杯）作为静态封面。
* **转化交互**：禁用 AI 教练弹出抽屉（消除时空错乱感）。
* **底部出口**：保留为 `[ Post My Result ]`，允许用户对历史高光时刻进行“延迟分享”或“考古发帖”。

### 3. 他人分享态 (`view_mode = public`)

* **场景**：其他用户通过社区动态或站外第三方平台（如 Facebook/Instagram 等）打开该分享链接。
* **主视觉与内容**：禁用动效。页面顶部不展示个人信息，直接以最纯粹的运动数据（火焰徽章+数据面板）作为第一视觉锚点。
* **隐私脱敏（法务红线）**：强制隐藏绝对卡路里数值（替换为 Lock 图标），强制锁定心率（HR）等敏感的展开图表。
* **全局悬浮转化仓 (Sticky Footer)**：页面最底部常驻悬浮吸底仓。
* **上层（社交背书）**：展示分享者的头像、昵称、用户等级（如 `Lv.5`）及 `[ + Follow ]` 关注按钮。
* **下层（动作号召）**：展示高亮的 `[ Download App to Challenge ]` 按钮。点击判断是否安装 App，已安装则唤起，未安装则跳转应用商店。

## 二、 页面信息拓扑 (倒金字塔结构)

页面摒弃原版复杂的平铺和 Tab 切换，采用“从感性到理性、由概括到深潜”的层级结构，并根据运动类型（室内/户外）智能分发：

### 层级 1：情绪主视觉 (Hero Section)

* 占据首屏核心位置，大字号展示成就总结（"First Workout Done!" 或 "New Personal Record!"）。

### 层级 2：锻炼来源前置 (Context Anchor)

* **室内组合课程 (3 Class Stack)**：废弃原方案的 Warm Up / Main / Cool Down 横向 Tab。重构为**“歌单模式 (Playlist Style)”**的折叠面板。默认展示包含主课程信息的组合卡片及总时间段；点击展开后，以列表形式展示子课程的封面、名称和独立时长。
* **常规单节课/自由训练**：展示对应的封面图和标题，增加具体的**时间戳**（如 `2026/03/10 18:30 - 19:10`）强化日记感。
* **户外路线 (Outdoor Map)**：若为户外跑/骑行，直接渲染专属的 **[地图+基础数据] 融合卡片**。卡片上半部分展示包含起终点的轨迹地图，下半部分用虚线分割展示基础总览数据。

### 层级 3：基础数据总览 (Overview)

* *注：户外路线不展示此独立模块，已融合至地图卡片下方。*
* 平铺本次运动的 3 个最核心全局数据：时长 (Time)、消耗热量 (Calories)、距离 (Distance)。

### 层级 4：训练分布条 (Training Split)

* **场景限制**：仅在室内包含器械切换/上下机切换，或自由训练时展示。户外运动不展示此模块。
* **表现形式**：引入一条彩色的时间进度条（如深蓝/红色双拼），直观展示 `Off-Equipment (无器械)` 与 `On-Equipment (有器械)` 的时长占比，将底层碎片化数据升维为直观的结构洞察。

### 层级 5：高阶数据渐进式折叠 (Advanced Metrics)

* **组件化封装 (MetricRow)**：将心率 (HR)、配速 (Pace) 等专业字段封装为独立的折叠卡片。
* **渐进式交互**：默认折叠，外层仅展示平均数值（如 `Avg 138 BPM`）；用户主动点击后，向下平滑展开 SVG 趋势图表（如折线图、柱状图）。
* **AI 洞察 (AI Insight)**：在展开的图表下方，基于后台数据模型，附加生成一句话的白话版运动评价（例如：*“✨ AI Insight: 你有 60% 的时间处于燃脂心率区间，耐力表现极佳！”*）。

## 三、 特殊交互声明

1. **AI 教练抽屉的关闭逻辑**：在即时态中，底部 AI 教练抽屉弹出后，用户必须显式点击右上角 `[ X ]` 或操作按钮关闭该抽屉，页面最底部的 `[ Post My Result ]` 按钮才会完整露出并可点击。
2. **组件复用要求**：H5 前端开发必须采用统一的高阶组件树（如 `<MetricRow />`），通过状态参数控制隐藏/脱敏/动效，严禁为三种业务状态开发三个完全独立的页面，以保障后续迭代的极低维护成本。

# 运动结果页 (Workout Result)

> **逻辑概述**：运动结果页为统一载体，通过 `view_mode` 参数实现一套代码、三种形态：练后即时态 (immediate/record)、历史查档态 (history)、他人分享态 (public)。页面采用倒金字塔信息结构：情绪主视觉 → 锻炼来源 → 基础数据 → 训练分布（仅室内）→ 高阶数据折叠卡片。

---

## 1. 核心架构：三态解耦 (View Mode)

| view_mode | 场景 | 动效 | AI 教练抽屉 | 底部出口 |
| :--- | :--- | :--- | :--- | :--- |
| immediate / record | 刚完成运动，从锻炼引擎退出 | 一镜到底 + 庆祝动效 | 动效结束后延迟弹出，强阻断 | 关闭抽屉后露出 Post My Result |
| history | 从历史记录/日历进入 | 禁用动效，直接静态数据 | 禁用 | Post My Result（可延迟分享） |
| public | 站外分享链接打开 | 禁用动效 | 不适用 | 吸底仓：分享者信息 + Follow + Download App to Challenge |

**📝 业务逻辑**：
* **即时态**：主视觉先展示火焰/奖杯徽章与成就描述，随后图标与描述上移缩小，数据面板滑出；若破纪录则数据项展示红色 BEST，主视觉为奖杯。动效结束后延迟弹出 AI 教练底部抽屉，根据破纪录/有无计划展示对应建议。
* **历史态**：无庆祝与一镜到底，直接展示静态数据面板与成就徽章；不弹出 AI 教练抽屉。
* **分享态**：顶部不展示个人信息，以运动数据为第一视觉；强制隐藏绝对卡路里（Lock 图标）、锁定心率等敏感展开图表；底部常驻吸底仓含分享者头像/昵称/等级、Follow、Download App 按钮。

### ⬅️ 顶栏 `[workout_result_header]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 页面载入 / 各态通用 | Global | 左侧关闭、右侧分享（public 态无分享按钮） |

**📝 业务逻辑**：
* **布局**：左侧关闭按钮、中间标题（public 为 "Workout Shared"，否则为 "Workout Summary"）、右侧分享按钮（仅非 public 展示）。
* **动作/路由**：关闭返回上一级；分享进入社区发帖流（带图片/文案校验）。

---

### 🌟 情绪主视觉 (Hero) `[workout_result_hero]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 即时态：入场后展示，随后上移缩小 | L1 核心 | 留屏，与数据面板过渡 |
| 历史/分享态：静态展示 | L1 核心 | 留屏 |

**📝 业务逻辑**：
* **文案与视觉**：成就总结大字号（如 "First Workout Done!" / "New Personal Record!" / "Fastest 5K!"）；配火焰或奖杯徽章；即时态有副标题成就描述，历史/分享态可无副标题。
* **状态控制**：即时态/破纪录态先全屏庆祝（含彩屑等），随后主视觉上移缩小、数据面板滑出；历史/分享态无动效，直接静态展示。

### 📜 数据面板滚动区 `[workout_result_data_panel]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 即时态动效结束后 / 历史·分享态直接展示 | L1 核心 | 留屏，内嵌锻炼来源/总览/分布条/高阶数据 |

**📝 业务逻辑**：
* **内容**：承载「锻炼来源 → 基础数据总览 → 训练分布条（仅室内）→ 高阶数据折叠卡片」的滚动区域；户外时仅展示地图卡片与高阶数据。
* **可见性**：即时态庆祝阶段该区域隐藏，动效结束后滑出展示。

### 📋 锻炼来源 (Context) — 室内歌单 `[workout_result_context_class_stack]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 室内组合课程时展示，点击展开/收起 | L1 核心 | 留屏，展开后展示子课程列表 |

**📝 业务逻辑**：
* **交互规则**：歌单模式折叠面板。默认展示主课程信息与总时间段；点击展开后以列表展示子课程封面、名称、独立时长。
* **文案**：组合卡片标题如 "3 Class Stack"，副标题为时间戳（如 "2026/03/10 18:30 - 19:10"）。
* **场景限制**：仅室内组合课程展示；单节课/自由训练展示单课封面与时间戳；户外不展示此模块。

### 🗺️ 锻炼来源 — 户外地图卡片 `[workout_result_context_map]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 户外跑/骑行时展示 | L1 核心 | 留屏 |

**📝 业务逻辑**：
* **布局**：上半部分为轨迹地图（起终点+路线），下半部分虚线分割展示基础总览（时长、热量、距离）。
* **隐私**：public 态下卡路里替换为 Lock 图标。
* **场景限制**：仅户外路线展示；户外不展示独立「基础数据总览」模块（已融合在地图卡片下）。

### 📊 基础数据总览 `[workout_result_overview]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 室内/单课时展示 | L1 核心 | 留屏 |

**📝 业务逻辑**：
* **内容**：平铺 3 项核心数据：时长 (Time)、消耗热量 (Calories)、距离 (Distance)。

### ❤️ 高阶数据 — 心率 `[workout_result_metric_hr]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 点击展开/收起 | L1 核心 | 留屏，展开显示趋势图与 AI Insight |

**📝 业务逻辑**：
* **交互规则**：默认折叠，仅展示平均数值（如 Avg 138 BPM）；点击后向下展开 SVG 趋势图；public 态锁定，展示 Lock + Hidden，不可展开。
* **AI 洞察**：展开后图表下方可展示一句白话版运动评价（如燃脂心率区间占比等）。

### 📤 底部 CTA — Post My Result `[workout_result_cta_post]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 即时/历史态：关闭 AI 抽屉后露出（即时态）或直接展示（历史态），点击 | L1 核心 | 进入社区发帖流（图片/文案校验） |

**📝 业务逻辑**：
* **状态控制**：即时态下 AI 教练抽屉弹出时覆盖底部按钮；用户必须显式点击抽屉右上角关闭后，Post My Result 才完整露出并可点击。
* **样式**：即时/破纪录态为主按钮（红底）；历史态可为描边样式（白底红边）。
* **场景限制**：仅非 public 态展示；public 态使用专属吸底仓。

### 🤖 AI 教练留存建议抽屉 `[workout_result_coach_sheet]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |
| :--- | :--- | :--- |
| 即时态动效结束后延迟弹出；用户点击关闭 | Global | 关闭后露出 Post My Result |

**📝 业务逻辑**：
* **触发条件**：仅 view_mode 为 immediate 或 record 时，在动效结束后延迟弹出底部抽屉。
* **内容**：根据破纪录/有无计划展示对应 AI 教练建议卡片与主行动按钮（如 "Get my personalize plan" / "Start Cooldown Stretch"）。
* **关闭逻辑**：用户点击右上角关闭或完成主行动后关闭；关闭前底部 Post My Result 被遮挡不可点。
* **场景限制**：历史态与分享态不弹出此抽屉。

### 🔗 分享态吸底转化仓 `[workout_result_public_footer]`

| 触发方式 | 🧩 模块层级 | 🔗 页面出口 |

**📝 业务逻辑**：
* **上层（社交背书）**：分享者头像、昵称、用户等级（如 Lv.5）、"+ Follow" 关注按钮。

* **场景限制**：仅 public 态展示。

## 2. 特殊交互与组件约束

* **组件复用**：H5 必须采用统一高阶组件树（如 MetricRow），通过状态参数控制隐藏/脱敏/动效，禁止为三态开发三套独立页面。
* **隐私红线**：public 态强制隐藏绝对卡路里数值（Lock 图标）、锁定心率等敏感展开图表。

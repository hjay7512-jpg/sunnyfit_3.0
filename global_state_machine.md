# SunnyFit User Onboarding V3.0 - 全局状态机与路由地图

> **文档说明：**
* ** 本文档为 SunnyFit 新用户流程的全局路由与状态流转说明。它定义了各个业务节点（Pages）的前置条件、核心动作与出口跳转。
* ** 该部分主要针对新用户安装应用后的首次功能体验路径做优化，结合AI 虚拟教练，围绕“用户完成首次锻炼”为目标进行流程优化，旨在通过合理的新手引导路径，快速对齐用户的潜在使用预期，提升后续留存。

## 🗺️ 阶段一：注册与意图判断 (Onboarding & Intent)
*结合我们现有用户来源情况，在注册后，增加意图判断逻辑，根据用户不同的使用意图，引导至不同的新手引导路径*


### 1. 意图枢纽 (Intent Select)
* **页面标识：** [🔗 intent_select](?page=intent_select)
* **前置状态：** 新用户下载并首次打开 App，完成账号注册。
* **核心动作：** 注册账号-> 完善隐私设置（国家地区/公英制）-> 设置昵称 -> 选择虚拟教练 ->提供“连接设备”、“寻找课程”、“随便逛逛”三个分流口。
* **出口状态：**
  * `Action: 连接设备` -> 跳转至设备连接流程 [🔗 device_connect](?page=device_connect)
  * `Action: 寻找课程` -> 跳转至计划创建流程 [🔗 plan_create](?page=plan_create)
  * `Action: 随便逛逛` -> 跳转至情绪推荐流程 [🔗 just_explore](?page=just_explore)

### 2. 硬件配对流 (Device Connect)
* **页面标识：** [🔗 device_connect](?page=device_connect)
* **前置状态：** 从意图枢纽选择连接硬件。
* **核心动作：** 权限判断-> 设备扫描-> 结果展示-> 连接结果-> 生理信息填写，以及异常状态引导（型号校验）。
* **出口状态：** 
  * `Action: 配对成功` -> 进入Free Mode完成首次锻炼体验
  * `Action: 无智能设备` -> 推荐同器械类型的入门内容，引导完成锻炼体验

### 3. AI 计划生成向导 (Plan Create)
* **页面标识：** [🔗 plan_create](?page=plan_create)
* **前置状态：** 用户希望获得个性化的锻炼指导。
* **核心动作：** 沉浸式收集目标-> 可用器材-> 运动水平 -> 训练时间-> 注意事项-> 生理信息 -> 生成阶段性计划-> 开始首次锻炼。
* **出口状态：**
  * `Action: 点击 "START NOW"` -> 进入今日训练建议，完成第一次训练

### 4. 游客体验模式 (Guest Explore)
* **页面标识：** [🔗 just_explore](?page=just_explore)
* **前置状态：** 用户没有明确的锻炼想法，处于观望状态。
* **核心动作：** 极轻量的心情摸底 (Mood Check)-> 根据用户状态推荐一些轻度锻炼-> 引导完善生理信息-> 进入首次锻炼。
* **出口状态：**
  * `Action: 点击 "Start Session"` -> 引导开启训练体验。
